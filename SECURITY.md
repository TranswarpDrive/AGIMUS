# Security Policy

## Supported Versions

Security fixes are applied to the latest code on `main`.

## Reporting A Vulnerability

Please do not open a public issue for security-sensitive reports.

Instead:

1. Use GitHub private vulnerability reporting if it is enabled for the repository.
2. If that is not available, contact the maintainer through GitHub before public disclosure.

When reporting, please include:

- a clear description of the issue
- affected files or flows
- reproduction steps
- impact assessment
- any suggested mitigation if you have one

We will try to acknowledge valid reports promptly and coordinate a fix before public disclosure.
